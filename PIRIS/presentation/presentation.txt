
Презентация: Оценка размера программного продукта (метод функциональных точек)
Слайд 1: Заголовок

Оценка размера программного продукта (метод функциональных точек)

Слайд 2: Определение

Метод функциональных точек (МФТ) – это метрика для оценки размера программного обеспечения, основанная на функциональности, которую оно предоставляет пользователям.

Слайд 3: Преимущества

Объективность: МФТ не зависит от языка программирования, архитектуры или технологии реализации.
Повторяемость: МФТ обеспечивает унифицированный подход к оценке, что позволяет сравнивать разные проекты.
Ранняя оценка: МФТ можно использовать на ранних стадиях разработки, когда еще нет подробного дизайна.
Прогнозирование трудозатрат: МФТ можно использовать для прогнозирования трудозатрат и сроков разработки.
Слайд 4: Недостатки

Сложность: МФТ может быть трудоемким для применения, особенно для сложных систем.
Субъективность: Интерпретация некоторых правил МФТ может быть субъективной.
Неточность: МФТ является приблизительной оценкой и не учитывает все факторы, влияющие на размер проекта.
Слайд 5: Типы функциональных точек

Простые функциональные точки (ПФТ): отражают функции, которые обрабатывают данные.
Сложные функциональные точки (СФТ): отражают функции, которые управляют интерфейсом пользователя, производительностью и безопасностью.
Слайд 6: Расчет МФТ

МФТ = ПФТ + СФТ

ПФТ = Σ (EIF * Fi)

СФТ = Σ (CFi * Aj)

Слайд 7: Факторы, влияющие на МФТ

Сложность алгоритмов
Объем данных
Сложность интерфейса
Требования к производительности
Требования к безопасности
Слайд 8: Примеры применения МФТ

Планирование проекта
Оценка трудозатрат
Прогнозирование сроков разработки
Сравнение разных вариантов реализации
Управление рисками
Слайд 9: Инструменты для МФТ

Function Point Counting (FPC)
COSMIC
NESMA
Mark II FPA
Слайд 10: Заключение

МФТ является ценным инструментом для оценки размера программного обеспечения. Он позволяет получить объективную и повторяемую оценку на ранних стадиях разработки.

Примечания:

Слайд 1:

Изображение: График, иллюстрирующий рост функциональных точек в зависимости от сложности проекта.
Слайд 2:

Примеры функциональных точек:
Добавление записи в базу данных
Поиск информации в базе данных
Генерация отчета
Слайд 3:

МФТ может использоваться для сравнения разных вариантов реализации, например, веб-приложения и мобильного приложения.
Слайд 4:

МФТ не является точной наукой, но он может дать ценную информацию о размере проекта.
Слайд 5:

ПФТ и СФТ – это два основных типа функциональных точек.
Слайд 6:

EIF – это фактор сложности элемента данных.
Fi – это количество элементов данных.
CFi – это фактор сложности функции.
Aj – это количество функций.
Слайд 7:

МФТ – это лишь один из факторов, который следует учитывать при планировании проекта.
Слайд 8:

МФТ может использоваться для управления рисками, например, риском задержки проекта.
Слайд 9:

Существует множество инструментов для автоматизации расчета МФТ.
Слайд 10:

МФТ – это полезный инструмент, но его следует использовать в сочетании с другими методами оценки.
